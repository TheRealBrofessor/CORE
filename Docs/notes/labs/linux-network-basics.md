# Lab: Linux + Networking Basics

## Objective
Confirm you can inspect interfaces, routes, DNS, and HTTP from a Linux host.

## Tasks
1) Show interfaces:
- `ip a`

2) Show routes:
- `ip r`

3) Resolve DNS:
- `dig +short example.com`

4) Test HTTP:
- `curl -I https://example.com`

5) Check listening ports:
- `ss -tulpn`

## Success criteria
You can explain:
- what IP you have
- your default route
- DNS resolution result
- HTTP response status
